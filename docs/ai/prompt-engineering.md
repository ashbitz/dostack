# Prompt Engineering aplicado al proyecto DoStack

## 1. Revisar la estructura del archivo principal

Prompt:

Actúa como un desarrollador frontend senior especializado en JavaScript. Revisa el archivo app.js de mi proyecto de gestión de tareas y analiza la estructura general del código. Señala posibles mejoras en organización, legibilidad o mantenimiento sin cambiar la funcionalidad actual.

Por qué funciona bien:

Define un rol claro y pide revisar aspectos concretos del código. Así la IA se centra en mejoras útiles y no da respuestas genéricas.

## 2. Explicar cómo funciona el guardado en localStorage

Prompt

Explícame paso a paso cómo funciona el sistema de guardado de tareas en localStorage en este archivo. Analiza especialmente las funciones saveTasks() y loadTasks() y describe qué hace cada parte del proceso.

Por qué funciona bien

Pide razonamiento paso a paso y menciona funciones concretas del proyecto, lo que hace que la explicación sea más clara y específica.

## 3. Refactorizar una función del proyecto

Prompt

Actúa como un revisor de código JavaScript. Analiza la función validateTaskInput() paso a paso, explica qué hace cada parte y propone una versión refactorizada que mejore la claridad del código sin cambiar su comportamiento.

Por qué funciona bien

Combina rol, análisis paso a paso y una restricción clara. Así la IA mejora el código sin modificar su funcionamiento.

## 4. Documentar una función

Prompt

Documenta la función addTaskToDOM() usando comentarios en formato JSDoc. Explica qué hace la función, qué parámetros recibe y cómo afecta al DOM dentro de la aplicación.

Por qué funciona bien

Indica el formato de salida y la función concreta a documentar, por lo que la respuesta suele ser más ordenada y útil.

## 5. Buscar posibles mejoras en el código

Prompt

Analiza el archivo app.js y detecta posibles duplicaciones de lógica o funciones que podrían simplificarse. Explica brevemente cada mejora sugerida.

Por qué funciona bien

Es un prompt directo que pide analizar el código buscando problemas concretos, lo que suele generar respuestas prácticas.

## 6. Explicar el sistema de filtros

Prompt

Explícame cómo funciona el sistema de filtros de tareas en este proyecto. Analiza especialmente las funciones applyFilter(), setCurrentFilter() e isTaskVisible() y descríbelo de forma sencilla como si estuviera aprendiendo JavaScript.

Por qué funciona bien

Indica el nivel de explicación y menciona funciones concretas, por lo que la IA adapta mejor la explicación.

## 7. Revisar la validación del formulario

Prompt

Actúa como un desarrollador frontend senior. Analiza la función validateTextField() y su uso dentro del formulario de creación de tareas. Sugiere mejoras para hacer la validación más clara o reutilizable sin cambiar su comportamiento actual.

Por qué funciona bien

Define un rol y una restricción clara. Esto hace que la respuesta se centre en mejoras sin romper la lógica del código.

## 8. Añadir comentarios a funciones complejas

Prompt

Pide documentar funciones específicas, lo que ayuda a entender partes del código que pueden ser más difíciles de seguir.

Prompt
